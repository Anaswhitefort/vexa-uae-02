# Skills

触ったことのあるプログラミング言語・フレームワークや技術を書いていきます。所感は適当なので実際の能力とかけ離れている場合があります。許してください。
言語・フレームワークの隣に使用歴を書きますが、これはコンスタントに触っていた期間を表します。また、2019 年 11 月 05 日現在からの期間であることをご了承ください。

-   Python: 2 年 ちょい<br />人生で初めて触ったプログラミングです。競技プログラミングをしたり、twtter bot を書いたり、インターンで機械学習的な何かをしていました。割と使える方だと思います。
-   C/C++: 6 ヶ月くらい<br />大学の講義で使いました。深淵に触れる前に離脱したのでそこまで闇を見ていません。割とプログラミングでものを作る難しさと楽しさを教えてくれた言語なので感謝しています。講義では CUI 上で動作するぷよぷよもどきを作成しました。結構楽しかったです。
-   HTML&CSS: 1 年ちょい<br />個人で勉強しました。割と苦手ではないほうかなと思います。サークルのホームページを作ったりしました。
-   React.js: 7 ヶ月ちょい<br/>個人で使っています。特にインターンとかで使ってるわけではないですが、自分が作ったものを後悔するならフロントはできた方がいいよな〜くらいの気持ちはあります。大体の処理はかけると思いますが、練度が低いのでまだまだ工数が多くかかります。目下改善したいポイントです。
-   Gatsby.js: 2 ヶ月<br />React 製の Blazing fast🔥 なフレームワークです。React でのサイト制作は,音に聞く nuxt とかより面倒ごとが多いな〜(webpack とか)と思っていたところに現れたので虜になりました。割と Gatsby の気持ちを理解し始めてきてはいますが、まだまだ途上です。サイト一つ作るくらいならできると思います。
-   Haskell: 2 ヶ月ちょい<br />講義で触りました。関数型言語に触れたことがなかった自分にとってとても新鮮で良い体験になりました。「Haskell の美しさを知っている人は人生に絶望することはない」、至言だと思います。簡単な四則演算パーサを書いたりしましたが、正直詳しくはありません。
-   R: 3 ヶ月ちょい<br />こちらも初めは講義で。基本的な構文は理解している...といいたい。あとインターンで案件用のコードを書いたりしました。
-   Laravel: 2 ヶ月<br />teamLab のサマーインターン課題で使用しました。それまでサーバサイドのコードを書いた経験がほぼ無かったので、割と苦しみましたが色々なことを教えてもらいました。MVC とかレイヤードアーキテクチャの考えは Laravel がベースになっています。
-   Rust: 3 ヶ月<br />Rust ってなんかカッコいいよなぁくらいの安直な気持ちで触り始めました。2019 年のお盆休みは、ずっと Rust で競技プログラミングをしていたと思います。例のごとく Twitter bot を書いたり、競プロ用ライブラリを整えたり、OAuth1.0 を自力でやったりしました。とても好きな言語ですがまだまだ Rust の気持ちを理解できていないと思うので、継続的に触っていくと思います。
-   Go: 1 ヶ月<br />teamLab サマーインターンで使用しました。その時は DB アクセスのある API を書きました。構文的な負担が少なくて、とても書きやすい言語だったので、何か Web アプリ作るならバックエンドは Go にしようかなと思っています。Gopher くんが可愛くて好きです。

ここから先は使用歴を省いて書いていきます。ある程度かじったことがある程度のものがメインです。
